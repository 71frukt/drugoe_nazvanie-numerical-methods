# Реализация и тестирование функции ln(x)

Проект представляет собой интерфейсную библиотеку с реализованной функцией рассчёта логарифма пониженной точности ($< 4.0$ ULP). Функция поддерживает *Floating point flags* и *errno*.
Функция реализует следующий интерфейс:
```cpp
template <std::floating_point T, size_t TailorDeg = sizeof(T)>
T dumb_math::logarithm::ln(T x)
```
Функция использует метод lookup-таблицы размером $2^8 = 256$ строк и полиномиальную аппроксимацию рядом Тейлора степени `TailorDeg`. Эмпирически под требования к функции подобрано соотношение `TailorDeg = sizeof(T)`.

### Общие сведения
Алгоритм генерации произвольной математической функции состоит из нескольких этапов:
1. Фильтрация аргумента
2. Редукция аргумента к интервалу, удобному для аппроксимации
3. Аппроксимация
    * Путь 1
        * Lookup table
        * Полиномиальная аппроксимация
    * Путь 2
        * Начальное приближение
        * Итеративные методы
4. Реконструкция результата

Погрешность алгоритма оценивается в единицах *ULP (unit in the last place)* &mdash; расстояние между двумя соседними представимыми числами, ближайшими к данному.

Дробные числа *T* $x$ с мантиссой в $mantissa\_len$ бит с одинаковой экспонентой $exp(x)$ распределяются вдоль одного отрезка длиной $2^{exp(x)}$.  
Количество этих чисел равно $2^{mantissa\_len}$.  
Тогда $ulp(x) = 2^{exp(x) - mantissa\_width}$

Если эталонная функция &mdash; $f(x)$, а тестируемая &mdash; $g(x)$, то ошибка равна 

$$ulp_{error}(x) = \frac{|f(x) - g(x)|}{ulp(x)}$$

## Описание алгоритма расчёта натурального логарифма

### Фильтрация
&emsp; Область определения &mdash; $E = [0, +\infty)$.  
&emsp; $ln(0) = -\infty$;  
&emsp; $ln(+\infty) = +\infty$  
&emsp; На других точках $E$ возвращается не-NaN значение.  
&emsp; На точках вне $E$ возвращается NaN.

### Редукция

Сначала нужно привести аргумент $x$ к каноническому виду.

* Нормализованное положительное $x$:  
$x$ = $m \cdot 2^{exp}$, $m \in [1.0, 2.0)$ &mdash; нормализованная мантисса, $exp$ &mdash; целочисленная экспонента.

* Денормализованное положительное $x$:  
$x = 0.000...1xx \cdot 2^{exp}$ &mdash; нормализуем мантиссу.  
Ищем первую значащую единицу после точки, сдвигаем биты мантиссы влево на $t$ бит, чтобы эта единица встала перед точкой. Т.о. мы умножили $x$ на $2^t$, поэтому делим $x$ на $t$: $exp$ -= $t$. 

Функции вычисления мантиссы, нормализованной мантиссы и экспоненты числа *T* $x$ определены в 
[*[get_floating_parts.hpp]*](source/dumb_math/include/dumb_math/common/get_floating_parts.hpp).

Итак, $x$ = $m \cdot 2^{exp}$.  
$\ln(x) = \ln(m \cdot 2^{exp}) = \ln(m) + exp \cdot \ln(2), m \in [1.0, 2.0)$  

### Аппроксимация

* #### Построение lookup-таблицы

$m \in [1.0, 2.0)$ &mdash; слишком много для аппроксимации, поэтому отрезок разбивается на $2^k$ подотрезков, в центре каждого из которых ставим опорную точку $x_i$. Достав $k$ ведущих бит из $m$, определяем номер $i$ отрезка, в котором лежит мантисса. Редуцированный аргумент $v$:  
$v = m \cdot \frac{1}{x_i} = 1 + r$, $r = m \cdot \frac{1}{x_i} - 1 \in [-\Delta, \Delta]$  
$\Delta = \frac{2^{-k}}{2} = 2^{-k-1}$, то есть $r$ тем ближе к нулю, чем больше $k$.

$$\ln(m) = \ln\left(v \cdot x_i\right) = \ln(x_i) + \ln(1 + r)$$

$$\ln(x) = exp \cdot \ln(2) + \ln(x_i) + poly\_approx(r)$$

Таким образом, для каждого $x_i$ таблица должна хранить точные значения $\frac{1}{x_i}$ и $\ln(x_i)$, и расчёт логарифма произвольного числа $x$ сведется к рассчёту $poly\_approx(r)$.

Сами точные $\frac{1}{x_i}$ и $\ln(x_i)$ рассчитываются единожды при создании библиотеки.
[*[lookup_table_builder]*](source/dumb_math/modules/logarithm/tools/lookup_table_builder)

$\ln(x)$ для $x_i \in [1; 2)$ рассчитывается через ряд Ареатангенса:

$$x = \frac{1 + z}{1 - z} \implies z = \frac{x - 1}{x + 1} \in [0, \frac{1}{3})$$
$$\ln(x) = 2 \sum_{k=0}^{\infty} \frac{1}{2k+1} \left( \frac{x-1}{x+1} \right)^{2k+1}$$
Аргумент лежит на меньшем полуинтервале, поэтому ряд сходится значительно быстрее, за что приходится платить операциями деления.  
Алгоритмически ряд считается до тех пор, пока следующий член не перестанет влиять на сумму. Фактически это означает, что член ряда по значению меньше ULP, сумма ряда имеет максимальную для этого метода точность.
[*[approx.hpp - LnArtgTailorBestAcc_0to2(T x)]*](source/dumb_math/modules/logarithm/include/logarithm/common/approx.hpp)

* #### Рассчёт полиномиальной аппроксимации
Полином рассчитывается через ряд Тейлора порядка, зависящего от точности типа *T*. В коэффициентах ряда Тейлора также есть константные дроби, которые предрассчитываются в compile-time. [*[tailor.hpp]*](source/dumb_math/modules/logarithm/include/logarithm//tailor.hpp)

* #### Катастрофическая потеря точности в окрестности 1
В окрестности единицы происходит вычитание между табличным значением $\ln(x_i)$ и суммой ряда Тейлора которые крайне близки по значению, что, как известно, приводит к катастрофической потере точности, поэтому в окрестности 1 рассчёт ведется с помощью ряда Ареатангенса определенного порядка. [*[approx.hpp -  LnArtgTailor_0to2(T x))]*](source/dumb_math/modules/logarithm/include/logarithm/common/approx.hpp)


### Реконструкция результата
$$\ln(x) = exp \cdot \ln(2) + \ln(x_i) + LnTailor(1 + r), |x - 1| > 0.15$$
$$\ln(x) = LnArtanh(x), |x - 1| <= 0.15$$
